使用中文回答问题

新增和修改接口的时候必须同步修改相关的接口文档，按照规范开发！

当你在处理 bug 问题时，从全局角度出发，不要只考虑局部最优而使用一些能快速解决问题的办法，你做出的决定应该充分考虑到代码的设计模式，尤其是可维护性

在添加新功能时，我如果要求保留旧版本，再进行保留，否则只保留新版本，并且要形成版本说明，避免后续开发出现版本混乱！

在修改时请你确保对于修改的内容已经获取了足够的上下文，避免出现代码库层面的不一致，删除或者修改功能一定要确保代码中所有的引用位置都进行了修改！！！

对于已经创建好的对用户展示结果的页面禁止随便修改，除非我允许之后你可以充分评估后再开始修改！

更改路由信息等逻辑方面的问题时不要修改页面元素排版等美工问题

写数据库时如果遇到数据表之间的关联`禁止使用物理外键机型强行约束`，你应该在代码层面使用`逻辑外键`去约束数据的一致性

如果输出 sql 脚本文件，放在 docs/sql/ 目录下，并且每一个必要的字段必须有注释，方便进行维护。除此之外你需要给每一个字段安排合理的字段类型，避免语法错误，同时避免因为数据类型的问题导致的溢出。

需要进行联网搜索（如 git 等中国大陆访问困难的网站时）、使用 docker 官方镜像苍鹭进行镜像拉取、模型下载或者进行 git 等操作之前先确认网络代理已经开启，开启的命令是：export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890 all_proxy=socks5://127.0.0.1:7890 这个命令是立即生效的，因此可以不用等待直接衔接下一步的推送等操作

在执行新功能的落地前你需要先进行技术架构，并推理最合理的方式是什么，架构完成之后先给我查看，确认一下，再进行最终的落地实现

这是一个 agentloop 项目，更多的需要 AI 来参与Runtime，禁止在选项不固定的时候使用枚举，除非我同意。你要更多的使用 AI 的能力进行项目的建构，比如让 AI 生成预选等，减少 fallback 的使用！

项目启动相关的操作使用 npm 脚本完成，以下是此项目的 npm 脚本以及其对应的作用：


| 命令                     | 功能                                             |
| ------------------------ | ------------------------------------------------ |
| npm start                | 同时启动后端(9090)、用户端(5170)、管理后台(5171) |
| npm run start:frontend   | 启动用户端前端，先清理 5170 端口                 |
| npm run start:admin      | 启动管理后台前端，先清理 5171 端口               |
| npm run start:backend    | 启动后端，先清理 9090 端口                       |
| npm run stop             | 停止所有服务                                     |
| npm run restart          | 重启所有服务（后端 + 用户端 + 管理后台）         |
| npm run restart:frontend | 重启用户端前端                                   |
| npm run restart:admin    | 重启管理后台前端                                 |
| npm run restart:backend  | 重启后端                                         |
