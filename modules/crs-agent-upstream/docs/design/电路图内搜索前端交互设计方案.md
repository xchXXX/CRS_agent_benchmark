# 电路图内搜索前端交互设计方案

## 1. 目标

前端不改变现有资料搜索主流程。

现有结果仍然展示文档列表，点击结果仍然进入整篇文档预览。图内搜索只作为结果增强：

```text
普通文档结果
  + 如果 body_search.status = hit
  + 展示“图内精确命中”入口
  + 用户主动展开后再看局部信息
```

## 2. 当前前端接入点

当前文档搜索结果在：

```text
frontend/user/src/App.tsx
  SearchResult
  handleChatResponse -> case 'documents'
  message.type === 'results' 渲染结果卡
```

当前文档预览在：

```text
frontend/user/src/components/DocumentViewer.tsx
```

限制：

```text
DocumentViewer 目前是 iframe。
如果 iframe 加载的是跨域页面，前端不能可靠控制内部滚动、缩放、坐标高亮。
所以“自动定位到坐标”不能直接在当前 iframe 上硬做。
```

## 3. 数据结构

前端 `SearchResult` 增加：

```ts
interface CircuitBodySearch {
  status: 'hit' | 'no_hit' | 'unsupported' | 'failed'
  reason?: string
  pdf_id?: string
  keyword?: string
  source_pdf_url?: string
  viewer_url_type?: string
  raw_hit_count?: number
  page_hit_count?: number
  more_hits_count?: number
  best_hit?: {
    hit_id: string
    page_index: number
    page_number: number
    snippet?: string
    highlight_boxes_px?: number[][]
    preview_image_url?: string
  }
}

interface SearchResult {
  ...
  body_search?: CircuitBodySearch
}
```

结果映射时透传：

```ts
body_search: doc.body_search
```

只处理：

```text
result.body_search?.status === 'hit'
```

其他状态前端不展示。

## 4. 结果卡设计

### 4.1 默认状态

无图内命中时保持现在样式：

```text
[排名] 文档标题
       标签
                         查看 >
```

### 4.2 有图内命中

在标题和标签之间增加一条轻量命中提示：

```text
[排名] 文档标题
       图内精确命中 · 第 513 页 · 涡轮增压器
       标签
                         查看 >
```

交互：

```text
点击结果卡主区域：仍然打开完整文档。
点击“图内精确命中”提示：展开局部命中面板。
```

不要让所有候选项默认展示图片，避免搜索结果列表变重。

## 5. 展开面板

展开面板放在当前结果卡内部，位于标签下方。

结构：

```text
┌────────────────────────────────────┐
│ 局部命中 · 第 513 页                │
│ 涡轮增压器执行器 ...                │
│                                    │
│ [局部图缩略图 / 无图时显示坐标占位] │
│                                    │
│ [查看完整电路图] [查看搜索部分]     │
└────────────────────────────────────┘
```

展示规则：

```text
有 preview_image_url：
  显示局部缩略图。

没有 preview_image_url：
  不请求大图。
  显示命中页码、摘要、命中数量。
  “查看搜索部分”可以先置灰或降级为打开完整文档。
```

按钮：

```text
查看完整电路图
  复用当前打开文档逻辑。

查看搜索部分
  目标态：进入可控预览器，并定位到 page_index + highlight_boxes_px。
  当前 iframe 阶段：如果没有可定位 URL，不强行实现坐标定位。
```

## 6. 文档预览设计

### 6.1 当前阶段

继续使用 `DocumentViewer`：

```text
打开完整文档
```

如果点击“查看搜索部分”，但当前文档 URL 不支持页码参数：

```text
打开完整文档，并在标题栏显示：已命中第 513 页
```

这只是用户提示，不承诺坐标定位。

### 6.2 下一阶段

新增受控图内预览器：

```text
CircuitDiagramHitViewer
```

能力：

```text
展示 page image
根据 highlight_boxes_px 画高亮框
支持缩放
支持拖拽
支持“打开完整文档”
```

只有当后端能提供页图或局部图时启用。

## 7. 状态管理

新增状态：

```ts
const [expandedBodyHits, setExpandedBodyHits] = useState<Record<string, boolean>>({})
```

key 建议：

```text
`${message.id}:${result.file_id}:${body_search.best_hit.hit_id}`
```

原因：

```text
同一轮搜索结果内稳定
分页切换后可恢复展开状态
不同消息之间不会冲突
```

## 8. 点击事件

当前结果卡整行有 `onClick`。

新增图内命中入口时必须阻止冒泡：

```ts
onClick={(event) => {
  event.stopPropagation()
  toggleBodyHit(...)
}}
```

否则用户点击展开会直接打开完整文档。

## 9. 样式建议

新增 class：

```text
.result-body-hit-trigger
.result-body-hit-panel
.result-body-hit-meta
.result-body-hit-thumb
.result-body-hit-actions
.result-body-hit-button
```

视觉方向：

```text
不做大卡片嵌套。
用细边框、浅背景、左侧强调线表达“这是当前结果的内部命中”。
颜色沿用现有 cyan/emerald，不引入新主色。
```

移动端：

```text
缩略图全宽，固定 16:9 或 4:3。
两个按钮横向放不下时纵向排列。
```

桌面端：

```text
缩略图最多 280px 宽。
右侧展示摘要和按钮。
```

## 10. 后端补充字段建议

为了实现局部图展示，后端后续最好补一个字段：

```json
{
  "body_search": {
    "status": "hit",
    "source_pdf_url": "http://static2.51gonggui.com/wps/file/xxx.pdf",
    "viewer_url_type": "raw_pdf",
    "best_hit": {
      "page_index": 512,
      "page_number": 513,
      "snippet": "涡轮增压器执行器 ...",
      "highlight_boxes_px": [[571, 500, 1151.625, 670]],
      "preview_image_url": "/chat/api/circuit-body-search/preview/xxx"
    }
  }
}
```

如果要做完整定位预览，还需要：

```json
{
  "viewer_target": {
    "type": "page_image",
    "page_index": 512,
    "highlight_boxes_px": [[571, 500, 1151.625, 670]]
  }
}
```

## 11. 实施顺序

1. `SearchResult` 增加 `body_search` 类型。
2. `documents` 响应映射透传 `doc.body_search`。
3. 结果卡增加“图内精确命中”提示。
4. 增加展开状态和展开面板。
5. 先实现“查看完整电路图”。
6. `preview_image_url` 有值时展示局部图。
7. 后端支持页图后，再实现 `CircuitDiagramHitViewer`。
8. 最后接入“查看搜索部分”的坐标级定位。

## 12. 暂不做

- 不默认给每个结果展示图片。
- 不在当前 iframe 内强行注入滚动或高亮。
- 不改变文档搜索结果排序。
- 不改变现有整篇文档打开方式。
- 不展示 `no_hit`、`unsupported`、`failed` 状态。
