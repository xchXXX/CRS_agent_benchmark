## 一、迁移背景

旧项目（doc_search）是一个生产级的汽车售后支持系统，采用 `ChatOrchestrator → IntentRouter → Handler` 的单跳编排架构。新项目（crs-agent）采用 Pydantic AI Agent Loop 架构，LLM 自主决定工具调用，取代旧的意图路由和 Handler 状态机。

本文档记录迁移的当前进度和剩余步骤。

### 关键约束

1. **旧项目不动** — 只从旧项目复制代码到新项目
2. **迁移业务能力，不迁移编排方式** — `ChatOrchestrator`、`IntentRouter`、`BaseHandler`、`SessionManager`、`LifecycleManager` 不搬
3. **文档搜索走外部接口** — 实际业务通过共轨之家（GGZJ）API 搜索文档，不走内部 MySQL FULLTEXT（保留作为降级方案）
4. **Qdrant 暂不迁移** — 向量检索后续有需要再补
5. **微信小程序不迁移** — 已废弃
6. **用户端和管理端前端保持一致** — 直接复制旧项目前端，适配新后端 API
7. **鉴权必须优先迁移** — 不迁就无法测试和使用任何真实场景

#### Step 2.4 — System Prompt 调优
