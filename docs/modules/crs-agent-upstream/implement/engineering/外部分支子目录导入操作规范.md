# 外部分支子目录导入操作规范

## 目的

本规范用于约束这类需求：

- “把另一个仓库的某个分支内容放到 `modules/crs-agent-upstream`”
- “只更新某个模块目录，不要影响当前仓库根目录和其他模块”

核心目标是：

- 只同步目标模块目录
- 不切换当前仓库整仓分支
- 不污染仓库根目录工作区
- 避免再次出现“无共同历史分支覆盖整仓工作区”的事故

## 禁止事项

遇到外部仓库、外部远端、无共同历史分支、或用户明确要求“只放到某个子目录”时，禁止直接在当前仓库执行：

- `git switch <external-branch>`
- `git checkout <external-branch>`
- `git pull <external-remote> <branch>` 后再整仓切换
- 任何会让当前仓库 `HEAD` 直接落到外部历史上的操作

特别是当命令输出出现 `warning: no common commits` 时，说明目标分支与当前仓库历史不一致，绝对不能继续做整仓切换。

## 正确流程

### 1. 确认边界

执行前必须先确认：

- 当前仓库根目录
- 目标模块目录
- 外部分支来源地址
- 本次是否只允许影响目标子目录

如果用户要求是“只放到 `modules/crs-agent-upstream`”，那么后续所有同步动作都只能以该目录为边界。

### 2. 临时拉取外部分支

必须把外部分支单独拉到临时目录，例如系统临时目录：

- 临时克隆：`git clone --depth 1 --branch <branch> <repo> <temp-dir>`
- 或归档/导出到临时目录

要求：

- 临时目录不得复用当前仓库根目录
- 禁止把外部分支直接 fetch 后切到当前仓库工作区

### 3. 子目录同步

把临时目录中的内容同步到目标模块目录时，必须满足：

- 目标路径先做绝对路径校验
- 只允许写入 `modules/crs-agent-upstream`
- 禁止写到仓库根目录其他位置

推荐方式：

- 用稳定的目录同步工具从临时目录同步到目标模块目录
- 优先排除 `.git`
- 本地运行产物目录按需排除，例如日志、缓存、`node_modules`、`.data`

### 4. 结果校验

同步完成后，必须检查：

- Git 变更范围是否只落在 `modules/crs-agent-upstream`
- 根目录和其他模块是否未被改动
- 目标模块的关键文件是否已更新到目标分支版本

## 推荐执行模板

1. 确认仓库根目录与目标目录。
2. 把外部分支克隆到临时目录。
3. 预演同步范围。
4. 只同步到 `modules/crs-agent-upstream`。
5. 检查 `git status -- modules/crs-agent-upstream`。
6. 确认根目录无额外改动。

## 本仓库适用结论

以后凡是用户提出：

- “把某外部分支内容放进 `modules/crs-agent-upstream`”
- “只更新模块目录，不动仓库根目录”

都必须按“临时克隆/导出 + 子目录同步”的方式处理，不得再使用整仓切分支方案。
