# 聊天记录导出与可视化运行指南

## 目标

将企业微信群聊天记录从远端 MySQL 导出为两套审阅材料：

- 静态 HTML：用于人工快速浏览，页面样式接近微信聊天记录
- 原始 JSON：用于证据回查，避免 HTML 展示过程丢字段

## 脚本位置

- 代码入口：`C:\projects\CRS_agent\benchmark\doc_search_bench\chat_export\export_wecom_chat_html.py`

## 数据源

- host: `139.196.163.235`
- port: `65025`
- database: `wxt`
- table: `ggzj_work_wx_chat_content`

脚本默认已经内置以上连接参数，必要时可通过命令行覆盖。

## 运行方式

在仓库根目录执行：

```powershell
python -X utf8 C:\projects\CRS_agent\benchmark\doc_search_bench\chat_export\export_wecom_chat_html.py
```

常用参数：

```powershell
python -X utf8 C:\projects\CRS_agent\benchmark\doc_search_bench\chat_export\export_wecom_chat_html.py `
  --page-size 400 `
  --room-limit 20 `
  --message-limit-per-room 2000
```

参数说明：

- `--page-size`：每页消息条数，默认 `400`
- `--room-limit`：只导出消息量排序后的前 N 个群
- `--message-limit-per-room`：每个群只导出前 N 条消息
- `--output-dir`：显式指定导出目录

## 输出目录

默认输出到：

`C:\projects\CRS_agent\benchmark\reports\chat_exports\<export_id>\`

目录结构：

```text
chat_exports/<export_id>/
  index.html
  manifest.json
  raw/
    room_summaries.json
  rooms/
    <room_slug>/
      messages.json
      page-001.html
      page-002.html
      ...
```

## 打开方式

优先打开：

- `index.html`：全量群聊索引页

然后从索引页进入单个群：

- `rooms/<room_slug>/page-001.html`

这是纯静态 HTML，不需要启动本地服务。

## 审阅建议

- 先按索引页查看群消息规模和时间范围
- 再进入具体群，按分页阅读
- 需要回查完整字段时，以同目录 `messages.json` 为准

## 当前设计约束

- `path_url`、图片、视频、小程序卡片会保留原始链接，但不保证可逆推出可靠文件名
- HTML 仅用于审阅，不应直接作为 benchmark gold 证据
- benchmark case 仍应以聊天文本或 `msgtype=file` 中明确文件名为准
