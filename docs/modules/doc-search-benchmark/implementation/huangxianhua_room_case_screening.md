## 用户侧字段抽取补充口径

- `initial_user_message` 只允许基于用户原始发言做轻度去称呼、去口头语改写，不能带入客服回复内容
- `known_items` 必须覆盖用户在同一 case 中所有已发送消息里的明确信息，包含后续补充消息
- `known_items` 允许吸收题图中可直接读出的厂牌、车型、系列、控制器铭牌、零件号、发动机型号等，但不允许写入图片中看不稳的内容
- 不允许把唯一标题中的正式资料名、客服推荐结果、或 gold 中的型号信息逆向写回用户侧字段
- 如果题图里能稳定读到具体型号或编号，则优先放入 `known_items`，不要只保留“电脑板”“资料”这类过泛描述

## 本轮明确剔除的情况

- 只有入口层级，没有最终文档标题
  - 例如只圈车型入口、模块入口、`原厂图` 图标

- 同一答复中出现多个候选标题
  - 例如 `这两个参考下`
  - 或同一截图里红框包含多个并列推荐项

- 只有方向性说明，没有文档名
  - 例如 `参考下`
  - `在后面几页`
  - `只有电路图，插头实物图在图文详情里`

- 图片里标题不清楚或无法稳定识别
  - 这类样本全部不进入 train
